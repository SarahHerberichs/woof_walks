##VERSIONING

Prochain travail :

    à partir de develop, creer une branche sur laquelle le travail va avancer :
  • - git checkout develop(se positionner sur bonne branche)
  • git checkout -b feature/display-walks

    • --> chaque fonctionnalité est développée dans sa propre branche (exemple display-walks ou user-crud...)

  • - faire les commit dans nouvelle branche tant que travail n'est pas mergable :
  • git add . puis git commit -m "Ajout de la fonctionnalité de display-walks"
  • - quand fonctionnalité terminée : git checkout develop puis git merge feature/display-walks

FAIRE TOUS LES TESTS

  • git checkout develop
  • git checkout -b release/v1.0.0
    si vraiment ok :

    mise en prod dans branche main avec tag!

  • git checkout main et git merge release/v1.0.0 et git push origin main
  • git tag -a v1.0.0 -m "Version 1.0.0 stable" git push origin v1.0.0

    • Puis fusion de branche dans develop

  • git checkout develop puis git merge release/v1.0.0 puis git push origin develop
